# SQL 优化分析 Prompt

使用场景：分析慢 SQL，给出优化方案。

---

## Prompt 内容

```
请分析以下 SQL 语句的性能问题，并给出优化方案。

分析维度：
1. 是否会走索引（结合表结构判断）
2. 是否存在全表扫描风险
3. JOIN 顺序是否合理（小表驱动大表）
4. 是否有深分页问题（LIMIT offset 过大）
5. 是否有函数导致索引失效
6. 是否有 SELECT * 可以优化为按需取字段

输出格式：
## 问题分析
- 问题1：...
- 问题2：...

## 优化后 SQL
```sql
-- 优化说明：...
SELECT ...
```

## 建议添加的索引
```sql
ALTER TABLE xxx ADD INDEX idx_xxx (...);
```

## 预期效果
从 X 秒 → X 毫秒（基于数据量估算）
```

---

## 使用方式

将慢 SQL 和表结构（可选）发给 Claude Code，使用此 Prompt。
